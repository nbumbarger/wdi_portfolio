PlacemarkStory.com

PlacemarkStory.com is the easiest way to create and share beautiful interactive map stories. Creating a map is as simple as defining a series of event locations using a unique and intuitive interface, which allows place to be defined interactively on a map, using an address or current location, or even automatically based on the location that images were captured. Points of interest can be assigned additional attributes such as descriptions, images, or hyperlinks, and PlacemarkStory transforms the data into a shareable interactive map hosted in a personalized gallery.

For my flagship General Assembly project, I felt it was important to create an application that drew from my interest in Geography while filling a gap in the market; a real product that people would enjoy using. In the process, I believe I demonstrated a deep understanding of the course's concepts by generalizing them to a point that the original lessons are unrecognizable in the finished product. Blog posts to a database became GeoJSON datapoints on a map, adding items to a todo list became an AJAX system for realtime updating, and all of the behaviors are implemented using code that I am every bit as proud of as the application's user-facing functionality.

Features:

  • Intuitive, powerful UI
  • Automatic location parsing
  • Asynchronous streaming
  • Responsive styles
  • Innovative guest system
  • Custom validation system
  • Advanced API integration
  • GeoJSON back end

IMG2MAP tool

IMG2MAP allows users to drag-and-drop images from the desktop to the browser window to automatically generate a map. Because most images today have location built into their metadata, the program is able to produce maps with virtually no user intervention.

This application was built as a standalone tool in order for me to practice asynchronous, single-page JavaScript development (AJAX) as well as experiment with desktop-to-browser interactions. However, the concept is intended to be integrated with the core PlacemarkStory platform as an enhancement.

Features:

  • Single-page application
  • Asynchronous JavaScript
  • Desktop-to-browser interactions
  • Automatic location parsing
  • Cloud-based image storage
  • Responsive styles
  • GeoJSON back end

Spatial USA Today

This group project was conceived of as a spatially-enabled remix of the USA Today landing page. In place of a large image illustrating the leading article, the reimagined landing page presents a map which plots location for the top twelve articles in each section.

Behind the scenes, the project queries USA Today's newsfeed API for metadata from the latest articles, according to which section is active in the navigation bar (News, Sports, Life etc.). If it finds new articles, it adds their metadata to a database and then accesses the full JSON for each article in order to acquire an image to display with the link. This information is used to build the article link columns which surround the leading image on USA Today's homepage, as well as to produce a map indicating the articles' simulated location.

Features:

  • Mapbox.js API
  • Multiple USA Today APIs
  • Integrated style
  • Intelligent database updating

Flashcards.rb

This project served as our introduction to HTML and Ruby on Rails, giving us two days to convert a previously completed, command-line driven Ruby assignment to a database-driven Rails application. The Flashcards topic was assigned, but we were given freedom to develop our own implementation. As it was our first web-based project, I used the opportunity to conduct a few early experiments with Rails and web standards. I diverged from the example of Rails as a blogging platform and presented the flashcards application as a game, with scoring and automatic cycling through categories. I also developed a database system in which deleted items could be rolled back, and experimented with CSS animations and glyph font packages.

By first deciding on the functionality I wanted, then figuring out how it could be implemented, I quickly became comfortable with Rails and learned to see it as more of a general purpose web-development tool than a rigid publishing platform.

Features:

  • Advanced database functionality
  • Glyph font packages
  • CSS animation

My Background

I see web development not as a career pivot but as another skill to add to my toolbox, as I strive to be the very best technologist I can be. Everything I have done prior to taking up web development remains a relevant part of my experience as a developer, and I am at my peak effectiveness in roles which combine online dissemination of geospatial data with strong back end analytical and programming capabilities.

My career in Geographic Information Science and satellite remote sensing prepared me to make the most out of everything General Assembly had to offer, and the skills I learned along the way will serve me well in any field where visual, interactive communication skills, analytical thinking, and software development ability are required.
